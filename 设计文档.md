## 1. 项目概述

### 1.1 项目背景
开发人员、运维人员、系统管理员等在日常工作中经常需要使用各种命令行指令。这些命令往往分散在不同的文档、笔记或临时记录中，难以统一管理和快速检索。本项目旨在提供一个现代化的快捷命令管理工具，帮助用户高效地存储、分类、检索和复用常用命令。

### 1.2 项目目标
- 提供美观现代的用户界面，类 Notion 风格
- 支持命令的分类管理与快速检索
- 实现数据的导入导出功能
- 提升命令复用效率

### 1.3 当前状态
- **开发状态**: 已完成 v5.1 版本
- **部署状态**: 本地运行（开发模式）
- **下一步计划**: 添加拖拽排序功能

### 2.2 功能详细说明

#### 2.2.1 命令管理
- **新增命令**：点击"新增"按钮，填写名称、命令内容、分类、描述、标签
- **编辑命令**：点击命令卡片上的"编辑"按钮
- **删除命令**：点击命令卡片上的"删除"按钮，确认后删除
- **命令字段**：
  - 名称（必填）：命令的简短标识
  - 命令内容（必填）：实际的命令文本
  - 分类（可选）：所属分类
  - 描述（可选）：命令用途说明
  - 标签（可选）：逗号分隔的关键词

#### 2.2.2 分类管理
- **新增分类**：点击侧边栏底部"新增分类"按钮
- **编辑分类**：鼠标悬停在分类上，点击编辑图标
- **删除分类**：鼠标悬停在分类上，点击删除图标，相关命令移至"未分类"
- **自定义颜色**：8种预设颜色可选

#### 2.2.3 搜索检索
- 支持全文搜索（名称、命令内容、描述、标签）
- 支持按分类筛选（点击侧边栏分类）
- 实时搜索，输入即筛选
- 快捷键 `/` 聚焦搜索框（待完善）

#### 2.2.4 一键复制
- 点击命令卡片上的"复制"按钮
- 点击命令内容区域也可复制
- 复制成功显示绿色 Toast 提示

#### 2.2.5 Excel 导入导出
- **导出**：点击"导入/导出" → 选择"Excel" → 点击"导出"
- **导入**：选择 .xlsx/.xls 文件，自动识别分类并创建
- 字段映射：名称、命令、分类、描述、标签

#### 2.2.6 数据库文件导入导出
- **导出**：点击"导入/导出" → 选择"数据库" → 点击"导出"，下载 .db 文件
- **导入**：选择 .db 文件，直接恢复所有数据（覆盖当前数据）
- 用途：完整备份、数据迁移

## 4. 技术方案

### 4.1 技术选型

| 类型 | 技术栈 | 版本 | 说明 |
|-----|-------|------|------|
| 前端框架 | Vue 3 | 3.4.x | Composition API |
| 构建工具 | Vite | 5.x | 快速构建 |
| 样式方案 | Tailwind CSS | 3.4.x | 原子化 CSS |
| 数据存储 | sql.js | 1.14.x | 浏览器端 SQLite |
| Excel处理 | SheetJS (xlsx) | 0.18.x | Excel 导入导出 |

### 4.2 数据库设计（SQLite）

采用极简设计，仅两张核心表：

#### 4.2.1 分类表 (categories)
```sql
CREATE TABLE categories (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL UNIQUE,
    color TEXT DEFAULT '#0066CC'
);
```

#### 4.2.2 命令表 (commands)
```sql
CREATE TABLE commands (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    content TEXT NOT NULL,
    category_id INTEGER,
    description TEXT,
    tags TEXT,              -- 逗号分隔的标签
    sort_order INTEGER DEFAULT 0,
    FOREIGN KEY (category_id) REFERENCES categories(id)
);
```

**设计说明**：
- 使用自增 INTEGER 作为主键，简单高效
- 标签存储为逗号分隔字符串，避免额外关联表
- sort_order 用于拖拽排序，数值越大越靠前

### 4.3 sql.js 加载方案

由于 sql.js npm 包的 ES Module 导入方式存在兼容性问题，采用以下方案：

1. **本地文件加载**：将 `sql-wasm-browser.js` 和 `sql-wasm-browser.wasm` 复制到 `public/` 目录
2. **HTML 引入**：在 `index.html` 中通过 `<script>` 标签引入
3. **全局对象**：使用 `window.initSqlJs` 初始化

```html
<!-- index.html -->
<script src="/sql-wasm-browser.js"></script>
```

```javascript
// database.js
SQL = await window.initSqlJs({
  locateFile: (file) => '/' + file
});
```

### 4.4 数据持久化策略

- 运行时数据保存在 sql.js 内存中
- 每次操作后自动导出二进制数据存入 localStorage
- 支持导出完整 .db 文件作为备份
- 导入 .db 文件可直接恢复所有数据

## 5. 用户使用指南

### 5.1 快速入门

#### 5.1.1 启动项目
```bash
cd quick_search
npm install
npm run dev
```
或双击 `启动项目.bat`

#### 5.1.2 添加第一个命令
1. 点击右上角"新增"按钮
2. 填写命令名称、内容
3. 选择分类（可选）
4. 点击"保存"

#### 5.1.3 创建分类
1. 点击侧边栏底部"新增分类"
2. 输入分类名称
3. 选择颜色
4. 点击保存

#### 5.1.4 搜索命令
- 在顶部搜索框输入关键词
- 支持搜索：名称、命令内容、描述、标签

#### 5.1.5 复制命令
- 点击命令卡片上的"复制"按钮
- 或直接点击命令内容区域

### 5.2 数据管理

#### 5.2.1 Excel 导出
1. 点击"导入/导出"
2. 选择"Excel"
3. 点击"导出为 Excel 文件"
4. 自动下载 commands.xlsx

#### 5.2.2 Excel 导入
1. 点击"导入/导出"
2. 选择"Excel"
3. 选择本地 .xlsx/.xls 文件
4. 自动导入，分类不存在会自动创建

#### 5.2.3 数据库备份
1. 点击"导入/导出"
2. 选择"数据库"
3. 点击"导出为数据库文件"
4. 下载 quick_commands.db

#### 5.2.4 数据库恢复
1. 点击"导入/导出"
2. 选择"数据库"
3. 选择 .db 文件
4. 数据恢复（覆盖当前数据）

#### 5.2.5 清空数据
- 在导入导出面板底部点击"清空所有数据"
- 确认后清空所有命令和分类
